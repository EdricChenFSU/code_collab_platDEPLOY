## Overview

AI-Assisted Collaborative Code Platform is a Flask-based collaborative coding workspace. Users can register, log in, create projects, organize files and folders, invite collaborators, edit project files, and run Python files directly from the browser.

The project currently focuses on GitHub-like project management, persistent collaboration access, a browser-based code editor, and project file operations.

---

## Current Features

### Accounts and Dashboard

- User registration and login
- Session handling with stale/deleted-user protection
- Dashboard with owned projects, shared projects, pending invites, and online users
- Active-user heartbeat display

### Projects

- Create projects from the dashboard
- Each project starts with a `README.md`
- Project files are stored in a real filesystem-style structure
- Create nested folders and files from the selected folder context
- Open, edit, save, revert, delete, and download project files
- Download selected folders as `.zip` archives
- Download the project root as a `.zip` archive named after the project

### Collaboration

- Search users by username while inviting collaborators
- Send project invitations to existing users
- Accept invitations from the dashboard
- Accepted collaborators keep access until the owner removes them
- Owners can view and remove collaborators
- Collaborators can open shared projects from the Shared Projects tab

### Workspace

- Monaco Editor-based code editor
- File tree with collapsible folders
- Contextual file/folder actions after selecting an item
- Unsaved-change warnings when navigating away
- Revert option for unsaved edits
- Read-only viewer state when another collaborator is actively editing
- Live editor updates over WebSockets

### Python Execution

- Run `.py` files from the workspace
- Automatically saves unsaved changes before running when possible
- Console panel displays stdout, stderr, timeout messages, and exit code
- Python execution works for both project owners and collaborators

### Deployment

- Dockerfile included for self-hosting with Coolify or similar platforms
- Persistent app data can be stored with `DATA_DIR=/data`
- Render configuration is included, though the app is better suited to an always-on server for live collaboration

## Tech Stack

### Backend

- Python
- Flask
- Flask-Sock for WebSocket routes
- SQLite for users and invitations
- bcrypt for password hashing
- Gunicorn for production serving

### Frontend

- Server-rendered HTML generated by Flask
- Vanilla JavaScript
- Monaco Editor
- CSS in `public/style.css`

### Storage

- SQLite database: `users.db`
- Project file storage: `accounts/<username>/projects/<project>/files/`
- Production/self-hosted data path controlled by `DATA_DIR`

## Running Locally

Create and activate a virtual environment, then install dependencies:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Start the app:

```bash
python3 app.py
```

Open:

```text
http://localhost:3000
```

---

## Docker / Coolify Deployment

The repository includes a `Dockerfile` for self-hosted deployment.

Important environment variables:

```env
PORT=3000
DATA_DIR=/data
SECRET_KEY=replace-with-a-long-random-secret
```

Attach persistent storage to:

```text
/data
```

The app stores runtime data there when `DATA_DIR=/data`:

- `/data/users.db`
- `/data/accounts/`

More notes are available in `SELF_HOSTING.md`.

---

## Important Notes

- Python code execution currently runs on the server with a timeout. This is acceptable for trusted testing, but it is not a secure sandbox for untrusted public users.
- SQLite and local file storage are simple and useful for this stage, but a larger production system would eventually need stronger isolation, backups, and possibly Postgres/object storage.
- Live collaboration uses in-memory WebSocket state, so running multiple server instances would require a shared realtime layer.
